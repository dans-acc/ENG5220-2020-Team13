Detailed Description

The passive monitor state child class.

The child class to MonitorState. It handles the functionality of what to do when the device is inside a fence, by dormantly monitoring the coordinates at which the device is at, so that in case of an event it can hand over to the ActiveMonitorState to handle notifications and further monitoring.

MonitorState * PassiveMonitorState::handleLatLng ( std::pair< double, double > &  latLng)
overridevirtual

Handles passive location monitoring; passively handled monitoring i.e. no notifications / storage.

When in the positive monitoring state, the current location is checked against the fences that are stored in account. If the device is out of a fence, the state is switched to the active monitoring state.

Parameters
latLngThe pair containing the latitude (first) and longitude (second) coordinates of the device.
Returns
Nullptr if state remains in passive, a pointer to ActiveMonitorState if the device should be monitored.
